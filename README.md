## Описание

REST API Микросервис, реализующий CRUDL операции над подписками и операцию подсчёта суммарной их стоимости.

## Функциональность

Реализована основная логика, описанная в ТЗ:
- `POST /subscriptions` - создание новой подписки;
- `GET /subscriptions` - получение подписки по её id;
- `DELETE /subscriptions` - удаление подписки по id;
- `PATCH /subscriptions` - обновление данных подписки по её id;
- `GET /subscriptions/list` - получение списка всех подписок с установкой лимита элементов на странице и номером страницы;
- `GET /subscriptions/total-cost` - получение суммарной стоимости всех подписок за определённый период с возможностью фильтрации по пользователю или сервису;
- `GET /swagger` - интерактивная swagger-документация.

## Технологический стек

### Backend
- **Язык:** Go
- **База данных:** PostgreSQL
- **Роутинг:** net/http
- **Пакет для работы с БД:** [sqlx](https://github.com/jmoiron/sqlx)
- **Пакет для логирования:** [slog](https://pkg.go.dev/log/slog)
- **Генерация swagger-документации:** [swaggo](https://github.com/swaggo/swag)
- **Конфигурация:** .env

### Инфраструктура
- **Контейнеризация:** Docker + Docker Compose

## Запуск

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/Artymka/effective-mobile-test-task.git
   cd effective-mobile-test-task
   ```

2. Настройте переменные окружения
    ```bash
    cp .env.template .env
    # ...отредактируйте .env при необходимости
    ```

3. Запустите сервисы
    ```bash
    docker compose up -d
    ```
    Приложение будет доступно по адресу http://localhost:8000

## Документация

Документация уже сгенерирована, файлы `swagger.json` и `swagger.yaml` находятся в дирекотрии `./docs`. Также по ручке `/swagger` доступна интерактивная документация swagger ui.
Для генерации документации запустите следующие команды:
```bash
# установите пакет swaggo
go install github.com/swaggo/swag/cmd/swag@latest

# перезапустите терминал и перейдите в директорию проекта
# для генерации документации введите:
swag init -g ./cmd/main.go
```
